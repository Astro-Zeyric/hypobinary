# Security Policy for HypoBinary

## Supported Versions

All versions of HypoBinary are experimental and designed for educational and entertainment purposes. There are no guarantees of security or safety in production environments. Use at your own risk.

## Reporting a Vulnerability

If you discover a vulnerability or bug that could compromise the application or encoding/decoding, please report it responsibly:

- **Email:** [zeyric@proton.me](mailto:zeyric@proton.me)
- **GitHub Issues:** [Open an issue](https://github.com/Astro-Zeyric/hypobinary/issues)

Please include:

- The version of HypoBinary you are using
- Steps to reproduce the issue
- Screenshots or code snippets if applicable

## Disclaimer

HypoBinary is intentionally minimal, quirky, and experimental. It is **not intended for secure encryption** or sensitive data handling. All users should treat the project as a learning and creative tool only.
